# native_sqlite_android

Android implementation of the `native_sqlite` plugin. Uses a native Kotlin plugin class (`NativeSqlitePlugin`) backed by Android's bundled SQLite.

This package is automatically selected when your Flutter app runs on Android — you do not add it to your `pubspec.yaml` directly.

---

## Features

- Full CRUD, transactions, and raw SQL via the platform channel
- WAL journal mode enabled by default for safe concurrent access from both Flutter and native threads
- Kotlin schema helpers generated by `native_sqlite_generator` for native-side database access

---

## Native code generation

Running `dart run native_sqlite_generator` produces Kotlin files in the path configured by `native_sqlite_config.yaml`:

```
UserSchema.kt            ← column-name constants and CREATE TABLE SQL
UserHelper.kt            ← insert/query helpers for direct Kotlin use
SchemaVersionManager.kt  ← reads and writes PRAGMA user_version
Migration_X_Y.kt         ← migration stub (addColumn, renameTable, migrateTableData helpers)
```

---

## Configuration (`native_sqlite_config.yaml`)

```yaml
native_sqlite:
  android:
    enabled: true
    output_path: 'android/app/src/main/kotlin/com/example/myapp/generated'
    package: 'com.example.myapp.generated'
    generate_helpers: true
```
