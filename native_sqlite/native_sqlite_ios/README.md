# native_sqlite_ios

iOS implementation of the `native_sqlite` plugin. Uses a native Swift plugin class (`NativeSqlitePlugin`) backed by iOS's system SQLite.

This package is automatically selected when your Flutter app runs on iOS — you do not add it to your `pubspec.yaml` directly.

---

## Features

- Full CRUD, transactions, and raw SQL via the platform channel
- WAL journal mode enabled by default for safe concurrent access from both Flutter and native threads
- Swift schema helpers generated by `native_sqlite_generator` for native-side database access

---

## Native code generation

Running `dart run native_sqlite_generator` produces Swift files in the path configured by `native_sqlite_config.yaml`:

```
UserSchema.swift            ← column-name constants and CREATE TABLE SQL
UserHelper.swift            ← insert/query helpers for direct Swift use
SchemaVersionManager.swift  ← reads and writes PRAGMA user_version
Migration_X_Y.swift         ← migration stub (addColumn, renameTable, migrateTableData helpers)
```

---

## Configuration (`native_sqlite_config.yaml`)

```yaml
native_sqlite:
  ios:
    enabled: true
    output_path: '../native_sqlite_ios/ios/Classes/Generated'
    generate_helpers: true
```
